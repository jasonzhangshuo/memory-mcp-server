# 使用示例

本文档提供 personal-memory 技能的使用示例，帮助理解如何在实际对话中使用。

## 示例 1: 查询目标

**用户**: "我的目标是什么？"

**AI 行为**:
1. 识别到"我的目标"触发词
2. 调用 `memory_search(query="目标", category="goal")`
3. 返回找到的目标信息

**预期结果**:
```
根据您的记忆，您的核心目标是：50岁退休。具体来说，是为50岁退休做好身体与精神的双重准备。不是'再赢一次'，而是'不再走错'。
```

## 示例 2: 保存新记忆

**用户**: "记住：我周二有禅修课"

**AI 行为**:
1. 识别到"记住"触发词
2. 调用 `memory_add(category="commitment", title="周二禅修课", content="我周二有禅修课")`
3. 确认保存成功

**预期结果**:
```
已记录：周二禅修课
```

## 示例 3: 历史引用

**用户**: "我们上次聊的戒糖，后来怎样了？"

**AI 行为**:
1. 识别到"上次聊的"触发词
2. 调用 `memory_search(query="戒糖")`
3. 如果找到相关记忆，基于记忆回答
4. 如果未找到，诚实告知"没有找到相关记录"

**预期结果（未找到）**:
```
我没有找到关于"戒糖"的相关记录。您能告诉我更多信息吗？
```

**预期结果（找到）**:
```
根据之前的记录，[基于找到的记忆内容回答]
```

## 示例 4: 空结果处理

**用户**: "我们讨论过量子计算吗？"

**AI 行为**:
1. 识别到"讨论过"触发词
2. 调用 `memory_search(query="量子计算")`
3. 返回空结果
4. 诚实告知未找到，不编造内容

**预期结果**:
```
没有找到关于"量子计算"的相关记录。
```

## 示例 6: 重要决定（需要确认）

**用户**: "我决定每天冥想10分钟"

**AI 行为**:
1. 识别到"我决定"触发词
2. 识别这是重要决定，需要确认
3. 询问用户："这是一个重要决定，是否要保存到记忆中？"
4. 用户确认后，调用 `memory_add(category="decision", ...)`

**预期结果**:
```
这是一个重要决定，是否要保存到记忆中？
[用户确认后]
已记录：每天冥想10分钟
```

**预期结果**:

## 最佳实践

### 对于 AI

1. **宁多勿漏**: 不确定是否需要调用？调用。空结果比漏掉好。
2. **先读后答**: 涉及历史信息，必须先调用工具，再回答。
3. **确认再写**: 重要决定/目标/承诺，需用户确认后再保存。
4. **诚实为本**: 没调用工具获取的信息，不要声称"记得"。

### 对于用户

1. **显式保存**: 使用"记住"、"记下来"等明确指令
2. **历史查询**: 使用"之前"、"上次"、"我们讨论过"等触发词
3. **目标查询**: 使用"我的目标"、"我的计划"等明确查询
4. **重要决定**: 表达重要决定时，AI 会询问是否保存

## 常见问题

**Q: 为什么 AI 没有调用工具？**
A: 可能的原因：
- 触发词不够明确
- Skill description 需要优化
- MCP Server 未正确配置

**Q: 搜索返回空结果怎么办？**
A: 这是正常的。AI 应该诚实告知"没有找到相关记录"，不要编造内容。

**Q: 如何确保重要信息被保存？**
A: 使用显式保存指令："记住：XXX" 或 "记下来：XXX"
